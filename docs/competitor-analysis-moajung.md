# 모아줌(moajung.com) 경쟁 분석 & moasearch 갭 분석

> 작성일: 2026-06-25 · 대상: [moajung.com](https://moajung.com/) 외 동종 서비스
> 목적: moasearch와 정확히 같은 아이디어(중고거래 통합검색)를 가진 선행 서비스를 해부하고,
> 현재 moasearch 코드 기준으로 무엇이 같고 무엇이 부족한지 사실 기반으로 정리한다.

---

## 0. TL;DR

- **같은 아이디어의 서비스가 이미 다수 존재한다.** 모아줌은 그중 완성도가 가장 높은 편이고, 꿀모·중고닷 등 경쟁자가 더 있다. **단순 "여러 마켓 통합검색"만으로는 차별화 불가.**
- **모아줌의 진짜 무기는 검색 기능이 아니라 SEO 콘텐츠**다. `/category/{모델}` 시세 페이지와 `/tips/*` 가이드로 구글 유입을 만든다. 통합검색 서비스는 광고 없이 운영되므로 **검색 유입 = 생존**.
- **moasearch는 핵심 검색 엔진과 가격알림 cron까지 이미 실제로 구현돼 있다.** 기능 깊이는 모아줌과 대등하거나 일부 앞선다(가격알림 발동 엔진 보유).
- **moasearch의 결정적 약점은 2가지:** (1) **SPA라 SEO에 불리**(콘텐츠/시세 랜딩 페이지 0개), (2) **로그인 필수 구조**로 진입장벽이 모아줌(무가입)보다 높음.
- 권고: 마켓을 더 늘리기 전에 **시세 페이지(SSR/사전생성) + 무가입 검색**을 먼저 확보할 것.

## 1. 경쟁 환경: 이미 레드오션

"국내 중고마켓 통합검색"은 신규 아이디어가 아니라 이미 여러 팀이 운영 중인 영역이다.

| 서비스 | URL | 통합 대상 | 특징 |
|---|---|---|---|
| **모아줌** | moajung.com | 당근·번개·중고나라·N플리마켓·헬로마켓·후르츠패밀리 (6) | 무가입, 지역필터·가격알림·시세·찜, SEO 콘텐츠 강함 |
| **중고닷(joongmo)** | joongmo.com | 당근·중고나라·번개·후르츠패밀리·헬로마켓 (5) | 가이드/FAQ/플랫폼소개, 면책조항 명시 |
| **꿀모** | ggulmo.com | "대한민국 중고 장터 매물 통합" | 매물 통합 노출형 |
| **무명** | moo-myung.com | (모바일 통합) | — |
| 개인 개발 사례 | 디시·클리앙 등 | 주로 당근 전지역 검색 | "당근 전지역 검색 사이트" 글 다수 |

> 시사점: **차별화는 "어디를 합치느냐"가 아니라 "검색 후 무엇을 더 주느냐(시세/알림/UX/유입)"에서 나온다.** moasearch가 노리는 4곳(번개·헬로·당근·중고나라)은 모아줌/중고닷 커버리지와 거의 완전히 겹친다.

## 2. 모아줌 심층 분석

### 2.1 포지셔닝
- title: `모아줌 — 중고거래 통합검색 | 당근마켓 번개장터 중고나라 한번에`
- description: `당근마켓·번개장터·중고나라·N플리마켓·헬로마켓·후르츠패밀리를 한 번에 검색하세요. 가입 없이 무료로 이용 가능합니다.`
- 핵심 카피: **"가입 없이 무료"** — 진입장벽 제거를 1순위 후크로 내세움.

### 2.2 통합 대상 (6곳)
당근마켓 · 번개장터 · 중고나라 · **N플리마켓** · 헬로마켓 · **후르츠패밀리**
→ moasearch 대비 **N플리마켓·후르츠패밀리 2곳을 더 커버**.

### 2.3 기능 인벤토리
- **통합검색** + 실시간 가격비교
- **지역 필터** — "전국 시도·시군구 단위로 내 동네 매물만" (셀링포인트로 전면에 내세움)
- **가격 필터 / 정렬** — 예산·최신순·가격순
- **가격 알림(푸시)** — "목표 가격 이하 상품이 올라오면 푸시 알림을 받을 수 있습니다"
- **카테고리별 시세** — 스마트폰/노트북/태블릿 등 모델별 시세 비교
- **찜 & 위시리스트** — "관심 상품을 저장하고 한눈에 관리"
- 추천 카테고리: 📱스마트폰 💻노트북 🖥️태블릿 🎮게임기 🎧오디오 🏠가전 👜패션 📷카메라

### 2.4 SEO·콘텐츠 전략 ★ (모아줌의 진짜 해자)
검색 서비스의 트래픽은 결국 구글 오가닉에서 나온다. 모아줌은 이를 위해 **콘텐츠 페이지를 양산**한다.

- **시세 랜딩 페이지** `/category/스마트폰`
  - 표 형식 모델별 시세: `아이폰 17 Pro 256GB | 120~135만원` (범위 = 최저~최고 암시)
  - 상태별 차등 설명: "S급(미개봉)은 110만원대, A급(사용감)은 85만원선"
  - 모델 나열: 아이폰 17 Pro/17/16 Pro/16/15 Pro, 갤럭시 S25 Ultra/S25/S24 (용량 병기)
  - "인기 검색 키워드" 섹션으로 롱테일(아이폰17/16/15, 갤럭시 S25 등) 흡수
- **가이드/팁** `/tips/*`
  - `/tips/selling-tips` (빨리 파는 법: 사진·제목·가격·설명·게시시간)
  - `/tips/fraud-prevention` (사기 예방), `/tips/price-negotiation` (가격 흥정), `/tips/direct-vs-delivery` (직거래 vs 택배)
  - 태그 기반 내부 링크(`#판매팁 #중고팔기 #사진잘찍기 #중고거래`)로 회유
- 타깃 키워드: "중고 물건 빨리 파는 법", "중고 스마트폰 시세", "아이폰 15 Pro 256GB" 같은 **구매의도 높은 롱테일**.

> **이게 핵심이다.** 검색 결과 페이지(동적)는 구글이 잘 못 긁는다. 모아줌은 **모델·카테고리별 정적 시세/가이드 페이지**로 색인 가능한 표면적을 키우고, 거기서 통합검색으로 유입을 흘려보낸다.

### 2.5 기술/아키텍처 (추정)
- 콘텐츠 페이지가 SEO에 최적화돼 있고 경로가 `/category/*`, `/tips/*`로 정적·SSR스럽다 → **SSR 또는 사전 렌더링(Next.js류) 가능성**. (메타에서 프레임워크 단서는 직접 확인되지 않음 — 추정.)
- 6개 마켓 통합은 공식 검색 API 부재 마켓이 많아 **비공식 내부 API/스크래핑** 방식일 것(moasearch와 동일한 회색지대).

### 2.6 수익모델 & 면책
- 페이지상 **광고·유료기능 명시 없음** (현재 미수익화 추정).
- 중고닷은 **면책 문구 명시**("거래 당사자가 아니며 책임은 각 플랫폼의 판매자·구매자에게") — 법적 리스크 회피 패턴. moasearch도 동일하게 두는 것을 권장.

## 3. moasearch 현재 상태 (코드 기반 사실)

> 추정이 아니라 레포 코드에서 확인한 내용.

### 3.1 아키텍처
- **프론트:** React 19 + Vite 8 + react-router-dom 7 — **SPA** (`assets.not_found_handling: "single-page-application"`).
- **검색 백엔드:** Cloudflare **Pages Function** [`functions/api/search.ts`](../functions/api/search.ts) — 프론트와 동일 도메인이라 CORS/env 불필요. 결과 180초 캐시.
- **인증·DB:** Supabase (`price_alerts`, `notifications` 테이블) + 카카오 소셜 로그인.
- **알림 엔진:** 별도 Cron 워커 [`moasearch-cron/`](../moasearch-cron/src/index.ts) — **30분마다**(`*/30 * * * *`) 동작.

### 3.2 어댑터별 수집 방식 ([search.ts](../functions/api/search.ts))
4곳을 `Promise.allSettled`로 병렬 호출, 한 곳이 실패해도 나머지는 유지.

| 마켓 | 수집 방식 | 페이징/정렬 | 취약점 |
|---|---|---|---|
| 번개장터 | 비공식 내부 JSON API (`api.bunjang.co.kr/.../find_v2.json`) | 페이지·**서버 정렬 지원**(order) | 가장 안정 |
| 헬로마켓 | SSR HTML의 `__NEXT_DATA__` 파싱 | 페이지 O, 정렬 X(클라 정렬) | 마크업 변경 시 깨짐 |
| 당근 | `/search/{kw}/` → `__remixContext` loaderData 재귀 파싱 | **1페이지만**(커서 미노출), 정렬 X | 페이징 불가 |
| 중고나라 | Next.js **RSC flight 스트림** 복원 후 items 추출 | 페이지 O, 정렬 X | **가장 깨지기 쉬움** |

### 3.3 기능 구현 현황

| 기능 | 상태 | 근거 |
|---|---|---|
| 4개 마켓 통합검색 | ✅ **실구현** | `functions/api/search.ts` 어댑터 4종 |
| 무한 스크롤 페이징 | ✅ 실구현 | `page`·`hasMore` |
| 정렬(최신/낮은가격/인기) | ✅ 실구현 | 번개만 서버정렬, 나머지 클라정렬 |
| 결과 캐시 | ✅ 실구현 | `caches.default`, TTL 180s |
| 가격 알림(규칙 저장) | ✅ 실구현 | `price_alerts` CRUD ([alerts.ts](../src/lib/alerts.ts)) |
| 가격 알림 **발동 엔진** | ✅ 실구현 | cron 워커가 검색→조건매칭→`notifications` 적재 |
| 알림 목록 화면 | ✅ 실구현 | [notifications.ts](../src/lib/notifications.ts), 페이지 |
| 찜/위시리스트 | ✅ 실구현 | `wishlist.tsx` (Supabase) |
| 카카오 로그인 | ✅ 실구현 | commit 477c9f7 |
| 상품 상세/딥링크 복원 | ✅ 실구현 | `useResolvedListing` |
| 홈 "지금 뜨는 매물" | ⚠️ **목업** | [data.ts](../src/data.ts) 하드코딩(`trending`) |
| 지역 필터 | ⚠️ **축소/제거됨** | 사이드바 지역·상태 필터 제거(commit 3ae9982) |
| **시세(카테고리) 페이지** | ❌ **없음** | 해당 라우트/페이지 부재 |
| **SEO 콘텐츠/가이드** | ❌ 없음 | `/tips` 류 없음 |
| 실제 푸시(OS/웹푸시) | ❌ 없음 | VAPID/web-push/FCM 흔적 없음 → **인앱 알림만** |

> 주의: 가격알림은 "규칙 저장+30분 cron 매칭+**인앱 알림 목록** 적재"까지다. **브라우저/OS 푸시는 아직 없다**(모아줌은 "푸시 알림"을 표방). 인앱 알림 → 웹푸시로 올리려면 VAPID 구독이 필요.

## 5. 갭 분석 — 모아줌엔 있고 moasearch엔 없는 것

1. **시세 랜딩 페이지(`/category/{모델}`)** — 가장 큰 전략적 격차. SEO 유입의 핵심.
2. **SEO 콘텐츠/가이드(`/tips/*`)** — 롱테일 키워드 흡수 + 내부링크 회유.
3. **SSR/사전렌더링** — moasearch는 SPA라 동적 결과가 색인되기 어렵다. 콘텐츠 페이지만이라도 정적 생성 필요.
4. **무가입 즉시 사용** — 모아줌의 1순위 카피. moasearch는 알림/찜에 로그인이 필요하지만, **검색 자체는 무로그인으로 열어야** 진입장벽이 낮아짐.
5. **마켓 커버리지 2곳(N플리마켓·후르츠패밀리)** — 우선순위는 낮음(차별화 요소가 아님).
6. **지역 필터 부활** — 모아줌이 전면에 내세우는 기능을 moasearch는 최근 제거했다.

## 6. 전략 권고 (우선순위)

### P0 — 생존(유입) 기반
- **시세 페이지를 사전 생성(SSG/SSR)으로 구축.** 카테고리×인기모델 표(평균/최저/상태별)를 cron으로 집계해 정적 페이지화. moasearch는 이미 검색 엔진 + 30분 cron + Supabase를 보유 → **시세 집계 파이프라인을 붙이기에 최적**. (검색 결과를 cron으로 모아 모델별 가격 분포를 저장 → 시세 페이지/그래프로 렌더.)
- **검색을 무로그인으로 개방.** 로그인은 알림·찜 같은 "저장형" 기능에만 요구.

### P1 — 차별화(모아줌 추월 포인트)
- **시세 추이 그래프 + 이상매물(시세 이하 급처) 탐지.** 모아줌은 정적 표 수준 → moasearch는 cron 자산으로 *시계열 시세*까지 가면 우위.
- **가격 알림을 실제 웹푸시(VAPID)로 승격.** 현재 인앱 알림 → 브라우저 푸시로 올리면 "푸시 알림" 표방 가능.
- **지역 필터 복구** — 마켓별 location 필드는 이미 수집 중(번개/당근/중고나라). 헬로만 매물별 지역 없음.

### P2 — 커버리지/위생
- N플리마켓·후르츠패밀리 어댑터 추가(차별화는 아니지만 "6곳" 패리티).
- 당근 1페이지 한계, 중고나라 RSC 취약성에 대한 **모니터링/알림**(어댑터가 조용히 0건 반환하면 감지).

## 7. 리스크

- **법적/약관:** 4개 마켓 모두 비공식 내부 API/스크래핑 의존(회색지대). 현재 코드 주석대로 "개인/포트폴리오 전제 + 공개 매물 정보만 + 캐시로 호출 최소화" 원칙 유지. **면책조항을 중고닷처럼 명시**(거래 비당사자) 권장.
- **기술 취약성:** 헬로(`__NEXT_DATA__`)·당근(`__remixContext`)·중고나라(RSC flight)는 **상대 마크업 변경 시 무음 실패** 가능 → 어댑터 헬스체크 필요.
- **차단 위험:** 트래픽 증가 시 비공식 엔드포인트 차단/레이트리밋 가능 → 캐시 TTL·요청 패턴 관리.
- **확장 한계:** 푸시·시세 집계로 트래픽이 늘수록 Pages Function 호출/Supabase 쓰기 비용 증가 — cron 주기·배치 크기 튜닝 필요.
